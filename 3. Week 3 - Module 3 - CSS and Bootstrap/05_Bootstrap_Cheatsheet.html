P3: Challenges with CSS – CSS的局限性 ★★

Q: What are the limitations of plain CSS? 纯CSS有哪些局限？
A: ① Centering pictures/text can be unintuitive 居中操作不直观 ② Hard to create responsive styles for different devices 难以为不同设备创建响应式样式 ③ Many sites don't need the full power of CSS 多数网站不需要CSS全部功能 ④ Producing attractive style is highly creative 创建美观样式是高度创造性活动
Q: What does "responsive" mean here? "响应式"指什么？
A: Styles that change depending on the device they are being viewed on. 样式能根据不同设备自动调整。类比：像水一样适应不同容器形状。

P4: Frameworks – 框架概述 ★★

Q: What are CSS frameworks? CSS框架是什么？
A: Open-source libraries for developing responsive web-applications. 用于开发响应式Web应用的开源库。
Q: How do frameworks work? 框架如何工作？
A: They use CSS and JavaScript to build style components in a reasonably aesthetic way with very little effort. 使用CSS+JS以极少工作量构建美观的样式组件。
Q: What is the big advantage? 最大优势是什么？
A: Normally designed to be responsive — one application looks good on mobile, tablets, and PCs with minimal effort. 天生响应式：一个应用在手机/平板/PC上都好看，工作量极小。

P5: Common Frameworks – 常见框架

Q: Name 4 common CSS frameworks. 列举4个常见CSS框架。
A: ① Bootstrap — powerful front-end framework 强大前端框架 ② Tailwind CSS — utility-first CSS framework 工具优先框架 ③ Semantic UI — user interface is the language of the web 语义化UI ④ Foundation — advanced responsive framework 高级响应式框架
Q: What is the main disadvantage of using a framework? 使用框架的主要缺点？
A: Websites built with the same framework will often appear visually similar. 使用同一框架的网站外观往往相似。

P7: Loading Bootstrap – 加载Bootstrap ★★

Q: How do you include Bootstrap in your page? 如何在页面中引入Bootstrap？
A: Two ways: ① Download and host the code yourself 下载自托管 ② Reference a Content Delivery Network (CDN) in the header CDN引用放在header中
Q: What files does Bootstrap consist of? Bootstrap由哪些文件组成？
A: A CSS file + a JavaScript file + optional theme files. 一个CSS文件 + 一个JS文件 + 可选主题文件。
<!-- 1.核心CSS (Latest compiled & minified CSS) -->
<link rel="stylesheet"
  href="https://cdn.../bootstrap.min.css"
  integrity="sha384-..."
  crossorigin="anonymous">
<!-- 2.可选主题 (Optional theme) -->
<link rel="stylesheet"
  href="https://cdn.../bootstrap-theme.min.css">
<!-- 3.核心JS (Latest compiled & minified JS) -->
<script src="https://cdn.../bootstrap.min.js"
  crossorigin="anonymous"></script>
💡 integrity = Subresource Integrity hash for security 子资源完整性哈希校验; crossorigin="anonymous" = no credentials sent for CORS 跨域不发凭证

P8: Your First Bootstrap Page – 第一个Bootstrap页面 ★★

Q: What meta tag is required for Bootstrap? Bootstrap需要什么meta标签？
A: The viewport meta tag setting width to device-width (for zooming and scaling). 设置视口宽度为设备宽度的viewport标签。
Q: What does the .container class do? .container类有什么作用？
A: It automatically sets margins that respond to the page size. 自动设置随页面大小响应的边距。
<!DOCTYPE html>
<html lang="en">
<head>
 <title>Bootstrap Example</title>
 <meta charset="utf-8">
 <meta name="viewport"        <!-- 必须 -->
  content="width=device-width,
  initial-scale=1">           <!-- 1:1缩放 -->
 <link rel="stylesheet"
  href="...bootstrap.min.css">  <!-- CSS -->
 <script src="...jquery.min.js"></script> <!-- jQuery -->
 <script src="...bootstrap.min.js"></script><!-- JS -->
</head>
<body>
 <div class="container">       <!-- 响应式容器 -->
  <h1>My First Bootstrap Page</h1>
  <p>Inside .container class.</p>
 </div>
</body></html>
⚠ 3 files needed 需要3个文件: bootstrap.min.css + jquery.min.js + bootstrap.min.js (Bootstrap 3 depends on jQuery jQuery是依赖)

P9: Bootstrap Container – 容器类型 ★★

Q: What are the 3 types of Bootstrap containers? 3种Bootstrap容器类型？
A: ① .container — responsive fixed-width; max-width adjusts at each breakpoint (xs,sm,md,lg,xl,xxl); centers content 响应式固定宽度，居中内容 ② .container-fluid — always 100% viewport width regardless of screen size 始终100%视口宽度 ③ .container-{breakpoint} — fluid (100%) until that breakpoint is reached, then fixed-width 到达指定断点前流式，之后固定宽度

P10–11: Grid System Basics – 网格系统基础 ★★

Q: How many columns per row in Bootstrap? Bootstrap每行有几列？
A: 12 columns. Every component/div spans a number of these columns. 12列。每个组件/div占据若干列。类比：一把12厘米的尺子，你可以划分为不同的段。
Q: Must columns add up to 12? 列数必须加起来等于12吗？
A: Yes, the spanned columns should always add up to 12. 是的，跨越的列数总和应为12。超过12的列会自动换行。
Q: What is the layout hierarchy? 布局层级结构？
A: .container → .row → .col-* (container 容器 → row 行 → columns 列)
Q: What does class col (with no number) do? 不带数字的col类有什么作用？
A: Creates equal-width columns across all devices. Columns are centered in the parent .container. 在所有设备上创建等宽列。列在父容器中居中。
<div class="container">
 <div class="row">             <!-- 行 -->
  <div class="col">1 of 2</div> <!-- 等宽50% -->
  <div class="col">2 of 2</div> <!-- 等宽50% -->
 </div>
 <div class="row">
  <div class="col">1 of 3</div> <!-- 等宽33% -->
  <div class="col">2 of 3</div>
  <div class="col">3 of 3</div>
 </div>
</div>

P12: Column Sizing – 指定列宽 ★★

Q: How to specify column width? 如何指定列宽？
A: Use col-{n} where n = number of 12-column slices. 用col-{n}指定占n个列。
Q: What does col-6 mean? col-6是什么意思？
A: Span 6 out of 12 columns = exactly half the row width (50%). 占6/12列 = 正好一半宽度。
Q: How to create uneven layout? 如何创建不等宽布局？
A: Use different numbers: col-4 (1/3) + col-8 (2/3). 用不同数字：col-4占1/3 + col-8占2/3。
<!-- 等宽布局 Equal: 50%+50% -->
<div class="row">
 <div class="col-6">Half</div>
 <div class="col-6">Half</div>
</div>
<!-- 不等宽 Uneven: 33%+67% -->
<div class="row">
 <div class="col-4">1/3</div>    <!-- 33.3% -->
 <div class="col-8">2/3</div>    <!-- 66.7% -->
</div>

P13: Responsive Grid – 响应式网格 ★★

Q: What does col-md-6 mean? col-md-6是什么意思？
A: On medium devices and larger (≥768px), each column occupies 50% of the row width → two-column layout. Below md, columns stack vertically at 100%. 中等及以上设备(≥768px)每列占50%，低于md时垂直堆叠。
Q: Can you combine breakpoint classes? 能否组合断点类？
A: Yes! col-sm-6 col-lg-3 = on small: 2 columns per row (6/12=50%); on large: 4 columns per row (3/12=25%). 可以！小设备2列/大设备4列。
<!-- md+: 两列 -->
<div class="row">
 <div class="col-md-6">1</div>
 <div class="col-md-6">2</div>
</div>
<!-- sm:2列, lg:4列 -->
<div class="row">
 <div class="col-sm-6 col-lg-3">1</div>
 <div class="col-sm-6 col-lg-3">2</div>
 <div class="col-sm-6 col-lg-3">3</div>
 <div class="col-sm-6 col-lg-3">4</div>
</div>

P14: Bootstrap Tables – 表格 ★★

Q: What does class .table do? .table类做什么？
A: Adds basic styling with horizontal lines and padding. 添加水平线和内边距的基本样式。
Q: What additional table classes exist? 还有哪些表格类？
A: .table-striped 斑马条纹行 | .table-bordered 全边框 | .table-hover 悬停高亮 | .table-condensed 紧凑间距
Q: How to color specific rows? 如何给特定行上色？
A: Apply context classes to <tr>: success(绿) danger(红) info(蓝) warning(黄) active(灰)
<table class="table table-striped">
 <tr class="success">  <!-- 绿色行 -->
  <td>Doe</td><td>john@..</td>
 </tr>
 <tr class="danger">...</tr>  <!-- 红色行 -->
</table>

P15: Glyphicons – 图标字体

Q: What are Glyphicons? Glyphicons是什么？
A: ~200 icons included with Bootstrap. Rendered as text, so they resize with headers etc. 约200个图标，以文本渲染，随字号缩放。
Q: What is the syntax? 语法是什么？
A: Apply a class to an empty span. 对空span应用类名。
<button class="btn btn-default">
 <span class="glyphicon
  glyphicon-search"></span> <!-- 搜索图标 -->
 Search
</button>

P16: Other Features – 其他功能

Q: What other widgets does Bootstrap support? Bootstrap还支持哪些组件？
A: Jumbotrons 大型展示区 | Modals 弹窗 | Form layouts 表单布局 | Carousels 轮播图 — make pages look professional with little effort 少量工作让页面专业化

Exam Practice 考试练习 ★★

Q1: .container vs .container-fluid 的区别？
A: container有固定max-width随断点变化并居中; container-fluid始终100%视口宽
Q2: col-sm-6 col-lg-3 在sm设备布局？
A: 6/12=50%, 每行2个元素
Q3: Why add up to 12?
A: 12-col grid; >12 wraps to next line 超12换行
Q4: 手机正确缩放需要什么？
A: <meta name="viewport" content="width=device-width, initial-scale=1">
Q5: Bootstrap需要加载几个文件？
A: 3个: bootstrap.css + jQuery.js + bootstrap.js
Q6: col-md-6在xs设备上如何？
A: 100%全宽竖向堆叠(md仅≥768px生效)

Summary 总结 (P1–16)

  • CSS frameworks solve responsive+aesthetics 框架解决响应式+美观
  • Bootstrap = CSS + JS + themes via CDN 通过CDN加载
  • viewport meta tag is essential 视口标签必须
  • .container / .container-fluid wraps content 容器包裹
  • 12-col grid: .row → .col-{bp}-{n} 12列网格
  • 6 breakpoints: xs/sm/md/lg/xl/xxl 六断点
  • Table: .table .table-striped etc. 表格类
  • Glyphicons: empty span + class 图标语法
  • Widgets: jumbotron/modal/carousel/forms 组件
