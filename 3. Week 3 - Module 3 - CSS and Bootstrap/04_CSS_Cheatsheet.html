★★ What is CSS 什么是CSS P3

CSS = Cascading Style Sheets 层叠样式表
▸ Stylesheet language for the web 网页样式语言
▸ Specifies presentation (layout & style) of markup 指定标记语言的展示
▸ Applied to HTML and any XML 可应用于HTML和XML
▸ Superseded HTML attrs mixing presentation with content 取代了混合表现与内容的HTML属性

Advantages of CSS CSS优势 P4

1. Speed: Stylesheet downloaded once 样式表只需下载一次
2. Maintainability: Centrally maintained 集中维护
3. Accessibility: Semantic HTML+CSS for assistive tech 语义化+辅助技术
4. Portability: Consistent across devices 跨设备一致
5. Reduced work: No per-element styling 无需逐个设定
6. Consistency: Uniform look and feel 统一外观

★★ Three Levels 三个层级 P6

Level 层级Scope 范围Location 位置
Inline 内联Single tag 单标签style="…"
Document 文档Whole doc 整页<style> in <head>
External 外部Multi docs 多页<link> in <head>
"Cascading" = styles cascade from multiple sources! 层叠=多来源样式叠加

Inline Styles 内联样式 P7

<p style="color:yellow; background:purple">
  I have impeccable style.
</p>
⚠ Defeats purpose of stylesheets! 违背样式表初衷！Only for debugging 仅用于调试

Document Styles 文档样式 P8

<head><style>
  selector {         /* 选择器 */
    property:value; /* 属性:值 */
  }
</style></head>

External Styles 外部样式 P9

<head>
  <link rel="stylesheet"
    type="text/css"
    href="style.css"></link>
</head>
Format same as <style> contents 格式与style内容相同

Q&A 练习 P6–19

Q: Which CSS level has narrowest scope? 哪个层级范围最窄？
A: Inline — single tag only. 内联——仅单个标签
Q: :hover vs ::first-letter?
A: :hover = pseudo-class (state); ::first-letter = pseudo-element (part). 伪类(状态) vs 伪元素(部分)

★★ Combinator Selectors 组合选择器 P20–24

CombinatorSyntaxMeaning 含义
Group 分组s1, s2OR: s1 or s2 匹配s1或s2
Descendant 后代s1 s2s2 inside s1 (any depth) 任意深度
Direct child 子代s1 > s2s2 directly under s1 直接子
Sibling 兄弟s1 ~ s2s2 after s1, same parent 同父后续
Adjacent 相邻s1 + s2s2 immediately after s1 紧邻

★★ Combinator Examples 组合示例 P20–24

/* 分组: h2和b都变红 */
h2, b {color:red}
/* 后代: p内所有b (任意深度) */
p b {color:red}
/* 直接子代: p的直接子b (仅一层) */
p>b {color:red}
/* 兄弟: h1后所有h2 */
h1~h2 {color:red}
/* 相邻: 紧跟h1的h2 */
h1+h2 {color:red}

★ Descendant vs Direct Child P21–22

/* <p><span><b>X</b></span><b>Y</b></p> */
p b   → X AND Y /* 后代:任意深度 */
p>b  → only Y  /* 子代:仅一层 */
💡 Descendant = any depth; Direct child = one level only 后代=任意深度; 子代=仅一层

★★ Conflict Resolution 冲突解决 P25–28

Three sources 三种来源:
▸ User agent: browser defaults 浏览器默认
▸ Author: developer's CSS 开发者样式
▸ User: user browser settings 用户设置
!important can override priority 可覆盖优先级

★★ Precedence Rules 优先级规则 P26

Step 1: Origin & Importance 来源与重要性 (high→low):
Transition → !important user-agent → !important user → !important author → normal author → normal user → normal user-agent

Step 2: Specificity 特异性 (if tied):
▸ Inline > stylesheet 内联 > 样式表
▸ Count: IDs → classes/attrs/pseudo-classes → types/pseudo-elements

Step 3: Order 顺序: Last rule wins 后写的胜出

★★ Specificity Example 特异性示例 P27–28

/* 对 <p id="para" class="paragraph"> */
p          {color:yellow} /* 0-0-1 */
.paragraph {color:red}    /* 0-1-0 */
#para      {color:pink}   /* 1-0-0 */
/* Result: pink (ID wins) */

/* 加 !important → yellow wins! */
p {color:yellow!important}
/* Beats inline! 甚至胜过内联 */

★★ CSS Layout 布局 P35–41

The Box Model 盒模型 P36–38

Every element = 4 nested boxes 每个元素=4层嵌套盒子: 
┌────── Margin 外边距 ──────┐
│ ┌──── Border 边框 ────┐   │
│ │ ┌── Padding 内边距──┐│   │
│ │ │  Content 内容    ││   │
│ │ └──────────────────┘│   │
│ └─────────────────────┘   │
└───────────────────────────┘
★ width/height = content area ONLY! 仅内容区!
★ Total width = content + padding×2 + border×2
Margin affects spacing but NOT element size 外边距不算入元素尺寸

★ Margin, Border & Padding P37–38

Margin 外边距: outside border, always transparent 边框外,始终透明
▸ margin, margin-top/right/bottom/left
Border 边框:
▸ border-style: none|dotted|dashed|solid…
▸ border-width: thin|medium|thick|px
▸ border-color: any colour
Padding 内边距: between border & content 边框与内容间
▸ padding, padding-top/right/bottom/left

★★ Exam Q&A 综合练习

Q1: <p id="x" class="y" style="color:green"> with #x{color:red} .y{color:blue} — what color?
A: Green. Inline > #id > .class 内联最高(无!important时)
Q2: p > b vs p b?
A: p>b = direct child only; p b = any depth 直接子代 vs 任意后代
Q3: width:200px, padding:10px, border:5px — total width?
A: 200+10×2+5×2 = 230px. Margin不算入!
Q4: h1~h2 vs h1+h2?
A: ~ = ALL sibling h2 after h1; + = ONLY immediately next h2 全部 vs 紧邻
Q5: Which position value removes element from flow but offsets from viewport?
A: fixed. 固定定位:相对视口,脱离文档流
Q6: CSS rule with !important on element selector beats inline style?
A: Yes! !important beats inline styles. !important胜过内联样式
Q7: Specificity of #nav .item p?
A: 1 ID + 1 class + 1 element = 1-1-1
Q8: position:relative vs absolute?
A: relative: offset from normal pos, stays in flow 偏移但保留占位; absolute: offset from positioned ancestor, removed from flow 相对祖先,脱离流
Q9: What does float:right do?
A: Element moves to right side, text wraps on left 元素移到右侧,文本在左环绕
Q10: 3 advantages of external stylesheets?
A: Speed(download once), Maintainability(central), Consistency(uniform look) 速度/可维护/一致性
