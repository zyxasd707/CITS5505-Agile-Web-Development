★★ DOM 概述 P2–5

Q: 什么是DOM？
DOM (Document Object Model) 是平台/语言无关的接口，允许程序动态访问和更新文档的内容、结构和样式。它是HTML文档与程序之间的API。
Q: DOM树结构是什么？
DOM将HTML文档描述为树结构，document节点反映HTML层次结构。每个HTML元素→树中一个节点。
Q: 为什么JS对Web特别有用？
客户端JS添加了对象/方法/属性的集合，通过绑定DOM实现：动态文档(dynamic documents)、客户端编程(client-side programming)。

★★ BOM 浏览器对象模型 P6–7

Q: BOM和DOM的区别？
DOM树只是BOM树的子部分。BOM包含浏览器执行环境节点，不特定于当前页面：浏览器类型(navigator)、用户历史(history)、Cookies、屏幕尺寸(screen)、地理位置(geolocation)、本地存储(localStorage)。BOM无固定标准，但有通用特性集。
// BOM层次结构 (P7)
window
 ├── navigator  // 浏览器信息
 ├── history    // 浏览历史
 ├── location   // 当前URL
 ├── screen     // 屏幕信息
 ├── console    // 控制台
 └── document   // DOM树根节点
      ├── forms[]
      ├── images[]
      └── links[]

★★ DOM中的元素对象 P9

Q: HTML元素如何映射到JS？
HTML元素→JS元素对象(element objects)。元素的属性(attributes)→对象的命名属性(named properties)。
// 例: <input type="text" name="address">
// 对应对象有两个属性:
// type属性 → 值"text"
// name属性 → 值"address"

访问元素对象三种方式：按类型和位置(type&position)、按名称(name)、按ID(id)

DOM遍历示例 P15

function countTags(n) {   // n是Node
  var numtags = 0;       // 初始化计数器
  if (n.nodeType == 1)   // 1=ELEMENT_NODE
    numtags++;            // 是元素则+1
  var children = n.childNodes; // 获取子节点
  for(var i=0;i<children.length;i++){
    numtags += countTags(children[i]);
  }                       // 递归遍历
  return numtags;
}
// 调用: countTags(document) → 整棵DOM树

★★ Cookie操作 P23

// 设置Cookie（含过期时间）
document.cookie = "username=John Doe; "
  + "expires=Thu,18 Dec 2025 12:00:00 UTC";

// setCookie函数
function setCookie(cname, cvalue, exdays){
  var d = new Date();       // 当前日期
  d.setTime(d.getTime()
    + (exdays*24*60*60*1000)); // 加天数
  var expires = "expires="
    + d.toUTCString();      // 转UTC
  document.cookie = cname + "="
    + cvalue + ";" + expires + ";path=/";
}

// getCookie函数
function getCookie(cname) {
  var name = cname + "=";
  var ca = decodeURIComponent(
    document.cookie).split(';');
  for(var i=0;i<ca.length;i++){
    var c = ca[i];
    while(c.charAt(0)==' ') c=c.substring(1);
    if(c.indexOf(name)==0)
      return c.substring(
        name.length, c.length);
  }
  return "";
}

★★ 事件驱动编程概念 P25–28

Q: 什么是事件驱动编程？
程序流程由用户操作(鼠标点击、按键)或消息决定的编程范式。不限于Web：硬件中断、多进程OS、分布式编程、Java监听器等。
Q: 事件循环的两部分？
1. 事件检测(event detection) 2. 事件处理(event handling)。浏览器自动实现事件循环，程序员只需关注事件处理。
Q: 浏览器如何处理事件？
浏览器监听事件(轮询或中断)→识别事件→调用已注册的事件监听器(event listener)。事件处理代码必须注册到特定事件。

★★ 三种注册事件处理器 P30–31

// 方法1: HTML事件属性 (不推荐)

  Greetings</button>
// 或调用函数:

// 方法2: 元素对象属性

  .onclick = myHandler;
// 注意：传引用myHandler，不是调用()

// ★★方法3: addEventListener (推荐!)

btn.addEventListener("click",
  myHandler, false);
// 参数: 事件类型, 处理函数, 是否捕获阶段
💡 addEventListener优点：①关注点分离 ②同一事件可加多个handler ③可循环批量注册

★★ 事件流 (3阶段) P32–33

// 事件流: 事件传递到DOM树各节点的顺序
Document → HTML → BODY → DIV → BUTTON(target)
// Phase 1 捕获: root→target (从上到下)
// Phase 2 目标: target节点处理器执行
// Phase 3 冒泡: target→root (从下到上,默认)

重要事件说明 P34

  • mouseover/mousemove/mouseout 鼠标进入/移动/离开
  • keydown 按键按下
  • focus/blur 元素获得/失去焦点

★★ jQuery链式调用 P44

Q: 什么是链式调用(chaining)？
大多数jQuery修改方法返回修改后的元素对象(而非返回void)，因此可以在一条语句中连续应用多个操作。
// 链式调用示例
$("li")
  .filter(".first, .last")  // 过滤出first/last类
  .css("color", "red")      // 设前景色
  .css("background", "blue"); // 设背景色
// 步骤: 选li→过滤→设色→设背景

★★ 典型考题 Q&A

Q1: addEventListener比onclick好在哪里？
①关注点分离(JS与HTML分离) ②同一事件可注册多个handler ③可循环批量添加
Q2: event.target和event.currentTarget区别？
target=事件实际发生的元素; currentTarget=handler绑定的元素。冒泡时两者可能不同。
Q3: 为什么客户端验证不够？
可通过DevTools删除验证代码或直接构造HTTP请求绕过。服务器必须再次验证。
Q4: jQuery中$("ul li:first")和$("ul li:first-child")区别？
:first只选第一个匹配的li; :first-child选每个ul的第一个li子元素。
Q5: jQuery链式调用的原理？
大多数jQuery方法返回修改后的jQuery对象(return this)，允许连续调用。
Q6: 事件冒泡阶段的默认行为是什么？
默认在冒泡阶段处理(子→父)。addEventListener第3参数false=冒泡。某些事件(load,blur,focus)不冒泡。
