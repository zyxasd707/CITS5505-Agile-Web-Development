What is Client-side JS & DOM? ★★

P2–4
📖 P2: JavaScript 的独特卖点 USP

客户端 JavaScript 添加了一组对象、方法和属性，让脚本能与 HTML 文档交互——实现动态文档和客户端编程。

Client-side JavaScript adds a collection of objects, methods and properties that allow scripts to interact with HTML documents — enabling dynamic documents and client-side programming.

这是通过绑定到 DOM (Document Object Model，文档对象模型) 实现的。

📖 P4: DOM 的定义

"DOM 是一个平台无关、语言无关的接口，允许程序动态访问和更新文档的内容、结构和样式。"

"The Document Object Model is a platform- and language-neutral interface that will allow programs and scripts to dynamically access and update the content, structure and style of documents."

DOM 规范描述了文档的抽象模型 (abstract model)，是 HTML 文档与程序之间的 API。数据表示为属性 (properties)，操作表示为方法 (methods)。

JS — 探索 document 对象 (可运行)
// document 是 DOM 的入口对象
// document is the entry point to the DOM
console.log(document.documentElement.tagName); // "HTML" — 根元素
console.log(document.head.tagName);             // "HEAD"
console.log(document.body.tagName);             // "BODY"
console.log(document.title);                     // 页面标题 page title
console.log(typeof document);                   // "object" — DOM 是 JS 对象
▶ 运行 Run
🧠Q: DOM 和 HTML 源文件有什么区别？
HTML 文件是静态文本；DOM 是浏览器解析后生成的活的内存对象树 (live object tree)。
用 JS 修改 DOM 会立即反映在页面上（浏览器重新渲染），但不会改变 HTML 源文件。

DOM API 描述一个树结构。"document" 节点反映 HTML 文档中的层级关系。

DOM API describes a tree structure. The "document" node reflects the hierarchy in the HTML document.

          /        \
     <head>       <body>
       |              |
   <title>       <table>
       |            /     \
  "A simple   <tr>    <tr>
   document"  /|\     /|\

🌐 P6–7: BOM 浏览器对象模型

DOM 树只是更大的 BOM 树的子部分。BOM 还包括浏览器环境信息：浏览器类型、用户历史、Cookies、屏幕大小、地理位置、本地存储等。

The DOM tree is just one subsection of a larger BOM tree. Unlike the DOM, the BOM is not supported by a fixed standard.

                   window ← BOM 根节点
      ┌───────┬────────┬─────────┬─────────┬──────────┐
  self/window document frames[] location history  navigator
   parent/top   │                                         │
                ├─ forms[]  images[]         plugins[]  mimeTypes[]
                ├─ links[]  anchors[]
                └─ all[]    layers[]
⚠️ P6：DOM 有 W3C 标准；BOM 没有固定标准，但浏览器普遍实现了通用特性。window 是最顶层对象，document = window.document。
🧠Q: window 和 document 的关系？DOM 和 BOM 的关系？
window = BOM 根对象（整个浏览器窗口）
document = DOM 根对象 = window.document（当前页面）
DOM tree ⊂ BOM tree（DOM 是 BOM 的子集）

📝 P15: countTags 递归遍历 (课件原版代码) ★★
JS — P15 课件原版 (可运行)
// 递归函数: 统计 DOM 子树中所有 Element 节点的数量
// Recursive: count all Element nodes in a DOM subtree
function countTags(n) {               // n is a Node 参数是节点
  var numtags = 0;                    // Initialize counter 初始化计数器
  if (n.nodeType == 1)                // nodeType==1 → Element 节点
    numtags++;                          // If Element, increment 是元素则+1

  var children = n.childNodes;          // Get all children 获取所有子节点
  for (var i = 0; i < children.length; i++) {
    numtags += countTags(children[i]);  // Recurse on each 递归累加
  }
  return numtags;                      // Return total 返回总数
}

// 使用: 传入 document 遍历整棵 DOM 树
// Usage: pass document to traverse entire tree
alert('This document has ' + countTags(document) + ' tags');
▶ 在本页面执行 countTags(document)
⚠️ 易错：childNodes 包含所有节点类型！firstChild 可能是空白 Text 节点，用 firstElementChild 更安全。
🧠Q: childNodes 和 children 的区别？
childNodes → NodeList，含所有类型 (Element, Text, Comment)
children → HTMLCollection，只含 Element
常用 nodeType: 1=Element, 3=Text, 8=Comment, 9=Document

JS — Cookie 辅助函数 (P23 课件原版)
// 设置 Cookie (P23)
function setCookie(cname, cvalue, exdays) {
  var d = new Date();                                // 当前日期
  d.setTime(d.getTime() + (exdays*24*60*60*1000));   // 加上过期天数
  var expires = "expires=" + d.toUTCString();          // UTC 字符串
  document.cookie = cname+"="+cvalue+";"+expires+";path=/";
}

// 读取 Cookie (P23)
function getCookie(cname) {
  var name = cname + "=";                              // 搜索的键名
  var ca = decodeURIComponent(document.cookie).split(';'); // 分割
  for(var i = 0; i < ca.length; i++) {
    var c = ca[i];
    while (c.charAt(0) == ' ') c = c.substring(1);   // 去前导空格
    if (c.indexOf(name) == 0)
      return c.substring(name.length, c.length);      // 返回值
  }
  return "";
}
JS — localStorage (P24 课件代码，可运行)

  localStorage.clickcount = Number(localStorage.clickcount) + 1; // ⚠️ 转 Number!

}

// ⚠️ 所有值自动转为 string! All values become strings!

🧠Q: Cookie 和 localStorage 最关键的区别？
Cookie 随每次 HTTP 请求自动发送到服务器；localStorage 仅在客户端。
用 Cookie → 服务器需要读的数据 (session ID)。
用 localStorage → 纯客户端数据 (偏好、草稿)。

⚡ P26–28: 概念与浏览器事件循环

事件驱动：程序流由用户动作或消息决定。浏览器替你实现了事件循环——你只写处理函数 (handler)。

事件发生时，调用注册的 event listener。但必须先注册！

💡 append vs after: append 添加子元素(内部末尾)；after 添加兄弟元素(外部后面)。

jQuery 链式调用 Chaining ★★

P44
🔗 P44: Chaining 原理

大多数 jQuery 修改方法返回被修改的对象本身，因此可以连续调用。这种风格叫 chaining (链式调用)。

Most jQuery functions that mutate an element object return that mutated element object instead of returning nothing.

JS — P44 课件原版代码
$("li")                                // 1. 选中所有 <li>
  .filter(".first, .last")              // 2. 过滤出 class=first 或 last
  .css("color", "red")                 // 3. 设置前景色
  .css("backgroundColor", "yellow");  // 4. 设置背景色

// 原理: 每个方法返回 jQuery 对象自身 (return this)
▶ 原生 JS 复现链式效果
🧠Q: 链式调用能工作的核心原理？一句话概括。
每个 jQuery 修改方法都 return this — 返回 jQuery 对象本身 → 可继续调用。
设计模式: fluent interface / method chaining。
🧠Q: 用原生 JS 写出 $("li").filter(".first").css("color","red") 的等价代码
document.querySelectorAll("li").forEach(function(li) {
  if (li.classList.contains("first")) {  // 过滤
    li.style.color = "red";              // 设置样式
  }
});
