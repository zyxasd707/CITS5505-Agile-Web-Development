P5: HTTP请求/响应结构 ★★

  • Client-server architecture（客户端-服务器架构）
  • 客户端发request→服务器返response→遗忘一切=无状态stateless
Request 请求结构： 
GET /doc/test.html HTTP/1.1  ← 请求行(方法+URL+版本)
Host: www.test101.com         ← 请求头(键值对)
Accept: image/gif, */*
Content-Length: 35
                              ← 空行分隔header/body
bookId=12345&author=Tan       ← 请求体(数据)
Response 响应结构： 
HTTP/1.1 200 OK             ← 状态行(版本+状态码+文本)
Server: Apache/1.3.29         ← 响应头
Content-Type: text/html
                              ← 空行
<h1>My Home page</h1>       ← 响应体(HTML等)
Q: HTTP为什么是"无状态"的？影响？
服务器不记得之前的请求。需cookie/session维持状态。

P6: HTTP请求方法 Methods ★★

Method用途幂等
GET获取数据(retrieve)✅
POST创建/修改数据(create/alter)❌
PUT替换数据(replace)，执行2次=1次✅
DELETE删除数据(delete)✅
⚠ HTTP method字段只是字符串，可自定义，只要客户端/服务器约定好含义即可。
Q: PUT vs POST？
PUT幂等(idempotent)：重复执行效果相同；POST不幂等：可能创建多条记录。

P7: 响应状态码 Status Codes ★★

类含义重点码
1xx信息性 Info100 Continue
2xx成功 Success200 OK, 201 Created, 204 No Content
3xx重定向 Redirect301 Moved, 304 Not Modified
4xx客户端错误400 Bad Req, 401 Unauth, 403 Forbidden, 404 Not Found
5xx服务器错误500 Internal Error
💡 口诀：2成功 3跳转 4你错 5我错

P9: 异步通信 Async Communication ★★

  • 发请求时不知道服务器何时/是否响应
  • JS是单线程(single-threaded)，按序执行
  • 但运行环境(浏览器)不是单线程→回调机制
// 异步函数模式
Object.asyncFn(params, callbackFn)
// callback接收(error, response)参数
Event Loop 事件循环： 
请求Requests → [EVENT LOOP] → 耗时操作
               (单线程JS)    (文件/数据库/计算)
                    ↑               ↓
            触发Callback ← 操作完成
回调示例(P9)： 
function big_Request(data, callback){
  var req = request_to_server(data); // 发请求
  callback(req.error, req.response); // 触发回调
}
big_Request(myData, function(error,data){
  if(error) alert(error);  // 有错弹窗
  else render(data);       // 成功渲染
});
Q: JS单线程如何实现"不阻塞"？
浏览器Event Loop：网络请求由底层线程处理，完成后回调进队列，主线程空闲时取出执行。

P11: 发送HTTP请求 Sending Request

GET（参数在URL上）： 
xhttp.open("GET",
  "demo.asp?fname=Henry&lname=Ford",
  true);             // true=异步
xhttp.send();           // GET无body
POST（数据在body中）： 
xhttp.open("POST", "test.asp", true);
xhttp.setRequestHeader("Content-type",
  "application/x-www-form-urlencoded");
xhttp.send("fname=Henry&lname=Ford");
Q: open()第3个参数？
是否异步：true=异步(不阻塞), false=同步(阻塞)。

P12: 接收响应 Receiving Response ★★

function loadDoc() {
  var xhttp = new XMLHttpRequest(); // 创建
  xhttp.onreadystatechange = function() {
    if(this.readyState==4      // 完成
     && this.status==200) {   // 成功
      document.getElementById("demo")
        .innerHTML=this.responseText;
    }                          // 写入DOM
  };
  xhttp.open("GET","info.txt",true);
  xhttp.send();                // 发送
}
⚠ 核心判断：readyState==4 && status==200

P16: Fetch API ★★ (2017新方法)

fetch("https://api.com/drinks") // 默认GET
  .then(resp => resp.json())   // 解析JSON
  .then(data => {              // 处理数据
    console.log(data);
  })
  .catch(err =>                // 捕获错误
    console.error(err));
Q: 为什么两个.then()？
第1个拿Response对象，.json()返回新Promise；第2个才拿实际数据。

P17: Fetch + POST配置

fetch('https://api.com/order', {
  method: 'POST',           // 方法
  headers: {
    'Accept':'application/json',
    'Content-Type':'application/json'
  },
  body: JSON.stringify(     // 必须转字符串!
    {quantity:1,drink:'Tea'})
});
⚠ body不能直接传JS对象，必须JSON.stringify()！

P21: AJAX 动机 Motivation ★★

  • AJAX = Asynchronous JavaScript And XML
  • 是一种方法/理念(approach)而非具体技术
  • 名字有误导：实际常用JSON而非XML
  • 2005年 Jesse James Garrett 提出
  • 核心：异步HTTP请求→接收结构化数据→JS解析+DOM局部更新
  • 每个AJAX请求=一次HTTP交换，异步不冻结环境
  • 服务器返回数据对象(XML/JSON)→融入当前页面
Q: AJAX核心价值？
不需重载整页就能更新部分内容！如Gmail收发邮件不刷新。
Q: AJAX一定用XML？
不一定。虽名含XML，但更常用JSON或纯文本。

P22: AJAX时序图 Sequence Diagram ★★

HTML页面    JavaScript   XHR对象       Server
  |           |           |            |
  |--事件---->|           |            |
  |           |--create-->|            |
  |           |--URL+回调->|            |
  |           |           |--GET请求-->|
  |           |           |            |(处理)
  |           |           |<--XML/JSON-|
  |           |<-调回调---|            |
  |<--更新DOM--|           |            |
AJAX完整7步流程：
  • ① 用户触发事件(如onkeyup)
  • ② JS创建XHR对象(或fetch)
  • ③ 配置URL+回调，发送请求
  • ④ 异步等待(JS不阻塞)
  • ⑤ 服务器返回JSON/XML
  • ⑥ 回调触发，JS解析数据
  • ⑦ DOM操作局部更新(不整页刷新)
Q: 为何用onkeyup而非提交按钮？
自动补全(autocomplete)场景：每敲一键就请求，实时获取建议。如Google搜索建议。

P24: JSON 数据格式 ★★

  • JSON = JavaScript Object Notation
  • 用JS语法存储数据(对象字面量)
  • 比XML更简洁(succinct)，原生支持数组
  • 不应包含函数
{"employees":[              // 对象含数组
  {"firstName":"John",      // 元素=对象
   "lastName":"Doe"},
  {"firstName":"Anna",
   "lastName":"Smith"}
]}
解析/序列化： 
var obj = JSON.parse(str);   // 字符串→对象
var s = JSON.stringify(obj);  // 对象→字符串
💡 解析后的JS对象可直接更新DOM。

XML vs JSON 对比 ★★

特性XMLJSON
语法<tag>val</tag>{"key":"val"}
简洁冗长(标签重复)★ 简洁
数组嵌套标签模拟★ 原生[]
可执行❌❌(不含函数)
JS解析较复杂★ JSON.parse()
Q: JSON键与JS对象键区别？
JSON键必须双引号；JS对象键可不加。JSON值不能是函数/undefined。

★★ 典型考题 Practice Questions

Q1: 哪些HTTP方法是幂等的？
GET, PUT, DELETE。POST不是。幂等=执行多次与一次效果相同。
Q2: readyState==4 && status==200各表示？
readyState==4：请求完成。status==200：服务器OK。两者同时满足才处理响应。
Q3: .then()链→async/await改写：
fetch("/api").then(r=>r.json())
  .then(d=>show(d)).catch(e=>log(e))
// ↓ 等价：
async function f(){
  try{
    const r=await fetch("/api");
    const d=await r.json();
    show(d);
  }catch(e){log(e);}
}
Q4: 用JSON表示两本书？
{"books":[{"title":"JS","year":2020},{"title":"CSS","year":2019}]}
Q5: fetch()默认HTTP方法？
GET。POST需指定method:'POST'。
Q6: AJAX vs 传统页面请求？
AJAX=异步+局部更新(不刷新)；传统=同步+整页重载。
Q7: JSON.parse vs JSON.stringify？
parse：字符串→对象。stringify：对象→字符串。
Q8: CORS由谁控制？
由提供原始页面的服务器通过响应头控制浏览器能访问哪些其他域。
Q9: 为什么fetch body必须stringify？
HTTP传输的是文本/字节流，JS对象不能直接传输，需序列化为JSON字符串。
Q10: XHR的readyState有哪5个值？
0=UNSENT未初始化 1=OPENED已连接 2=HEADERS_RECEIVED已接收 3=LOADING处理中 4=DONE完成。
